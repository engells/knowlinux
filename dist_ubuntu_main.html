安裝方式

原則上用光碟或隨身碟方式開機，然後載入安裝專用的內核以及虛擬檔案系統，接著就進入安裝程序。安裝程序或者是在網路上下載套件，或者是以分區上 的 iso 檔案作為套件來源。在 Ubuntu 官網上可下載前述的內核以及虛擬檔案系統，搭配自行設置的可開機光碟或隨身碟，即可完成安裝用媒介。
可開機隨身碟的製作方式：
  • 切分隨身碟，第一分區設為可開機分區，並覆寫隨身碟 MBR
    fdisk /dev/sdX
    apt-get install mbr; install-mbr /dev/sdX
  • 安裝開機管理程式
    syslinux 或 grub
  • 放入安裝程式
各種安裝方式，採用 syslinux 系列之 boot loader：
  • 隨身碟兼作工具碟以及安裝碟，不同版本的 linux 可共用分區者，集中到第一個分區，其餘的 linux 各自放入獨立分區。
    詳細資料參閱 /home/engells/ktws/0_linux_docs/prog_boot_loader/syslinux-guide.txt
  • 隨身碟安裝官網的 initrd.gz 以及 vmlinuz
    • 自 http://archive.ubuntu.com/ubuntu/dists/版本代號/main/installer-平台種類 /current/images/hd-media/ 下載 initrd.gz 以及 vmlinuz
    • 自 http://releases.ubuntu.com/12.04.1/ 下載 ubuntu-12.04.1-alternate-amd64.iso
    • 將 initrd.gz 以及 vmlinuz 複製到可開機隨身碟
    • 編輯 syslinux.cfg 增加開機選項：
      default vmlinuz ← 若採網路安裝，核心為檔名 linux 的檔案
      append initrd=initrd.gz 或 append initrd=initrd.gz video=vesa:ywrap,mtrr vga=788 ← 後者為啟用圖形界面參數
    • 將 ubuntu-12.04.1-alternate-amd64.iso 複製到 /path/to/mount，也就是複製到 /dev/sdX；
    • 實作經驗，除了抓不到 iso 檔案外，其餘有正常開機。可能是兩者的內核版本不同所致。
  • 將官網的 boot.img 映像檔複寫至隨身碟，製成可開機隨身碟安裝
    • 準備好可開機隨身碟，第一區分區保留 800MB 或以上的容量。注意，第一分區必須設定 bootable 標誌。
    • 自 http://archive.ubuntu.com/ubuntu/dists/precise/main/installer-amd64/current/images/netboot/ 下載 boot.img.gz
    • 將 boot.img.gz 複寫至開機分區：zcat boot.img.gz /dev/sdX1
    • 經分析，boot.img 檔案是 syslinux 開機方式的映像檔，預設分區大小為 783MB，實際使用約 35MB。
    • 實作經驗，可順利開機。
  • 將官網的 mini.iso 燒錄為可開機光碟安裝
    • 自 http://archive.ubuntu.com/ubuntu/dists/precise/main/installer-amd64/current/images/netboot/ 下載 mini.iso
    • 燒錄 mini.iso 或 複製到 usb 隨身碟
    • 開機後從 isolinux 選單選取安裝方式，預設安裝基本系統。
    • 實作經驗，可順利開機。

採用 Alternate 模式安裝

  • 依照一般方式，將 Alternate 光碟複製到 USB-Flash 等裝置的分割區。
  • 以 syslinux 或 grub 導引開機後，按步驟進行，直到安裝程式回報找不到光碟時，退出掛載光碟畫面，回到上一層選單。
  • 按下「Alt + F2」組合鍵，在按下「Enter」 鍵，進入終端機。
  • 在終端機將含有 Alternate 光碟內容的分割區掛載到 「/cdrom」。
  • 按下「Alt + F1」組合鍵，回到主安裝畫面，進入掛載光碟選項。此時安裝程式應可取得光碟內容資訊。
    注意，若分割區是 ext2，直接用 mount /dev/sdXY /cdrom，不需要指定 -t ext2 參數。
  • 若採用 Alternate 光碟檔開機方式，則需將光碟檔 (***.iso) 掛載到 「/cdrom」。
  • 按照上述模式，可推測 Ubuntu 開機光碟或隨身碟開機後，Linux 核心會將光碟機內的光碟片掛載到 「/cdrom」，以進行後續安裝作業。

X視窗崩潰處理

  • tty
    以Ctrl + Alt + [F1~F6] 登錄tty
    top
    pkill / kill / killall
  • Ctrl + Alt + BackSpace
    System > Preferences > Keyboard > Layouts(T)  > Options(B) > Key sequence to kill the X server > Control + Alt + Backspace
  • Alt+sysrq+{reisub}
    系统启动时执行：echo "1" > /proc/sys/kernel/sysrq 或者修改 /etc/sysctl.conf 文件，设置 kernel.sysrq = 1
    /etc/sysctl.conf 檔案在 Ubuntu 中被移至 /etc/sysctl.d/ 目錄中，且切成數個子檔案，有關 kernel.sysrq = 1 的設定在 10-console-messages.conf 檔案

Ubuntu開機自動執行

  • 開機自動執行的程式有2種類型
    xwindow登入後自動執行 ←白話一點說就是：進入桌面後才自動執行
    ubuntu開機後自動執行    ←白話一點說就是：系統一開機就自動執行
  • xwindow登入後自動執行
    System (M) > Preference (M) > Starup Applications (M) > Startup Programs (T)，自行新增即可
    請參閱 ~/ktws/0_linux_doc/contents/gnome-xdg.html，有關 XDG 章節
  • ubuntu開機後自動執行
    有下列幾種方式
    • 編輯 /etc/init.d/rcS ，在檔案最後面添加指令
    • 編輯 /etc/rc.local或 /etc/init.d/rc.local，在檔案最後面添加指令
      例如在/etc/init.d/rc.local尾端寫入後續指令bash /opt/security/iptables/iptables.rule
    • 編寫 script 檔後，放到 /etc/init.d目錄下，如果在 /etc/init.d 中加入一個 script，例如lampp，當作服務來啟動
      手動啟用: sudo /etc/init.d/lampp start 或 sudo invoke-rc.d lampp start
      手動關閉: sudo /etc/init.d/lampp stop 或 sudo invoke-rc.d lampp stop
    • 使用update-rc.d 指令

網路設定

  • 在同一張卡上多設定一組IP
    sudo gedit /etc/network/interfaces
    auto lo
    iface lo inet loopback

    auto eth0 : 0
    iface eth0:0 inet static
    address 192.168.61.195
    netmask 255.255.255.0
    gateway 192.168.61.246

    auto eth0 : 1
    iface eth0:1 inet static
    address 192.168.61.194
    netmask 255.255.255.0
    gateway 192.168.61.246

  • 設定DNS
    sudo gedit /etc/resolv.conf
    nameserver 168.95.192.1
    nameserver 168.95.1.1

  • 設定NAT
    sudo vim /etc/sysctl.d/10-network-security.conf
    將其中 net.ipv4.ip_forward=0 改成 net.ipv4.ip_forward=1,下次開機時便自動設定完成了
    將其中 net.ipv4.conf.all.rp_filter=0 改成 net.ipv4.conf.all.rp_filter=1,下次開機時便自動設定完成了
    iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

Tips

  • 清除回收筒內無法刪除的檔案
    cd ~/.local/share/Trash && sudo rm -rf *
    cd /media/sdc1/.Trash-1000 && sudo rm -rf *
  • 將/bin/sh/bin/sh連結從/bin/dash，恢復為/bin/bash
    sudo dpkg-reconfigure dash
  • 刪除特定目錄下的空目錄
    find /tmp -depth -empty -type d
    find /tmp -depth -empty -type d -exec rmdir -v {} \;
  • 刪除NTFS檔案複製到Ubuntu時產生之多餘檔案
    find . -name "*encryptable*" -exec rm {} \;
    find . -name "*Identifier*" -exec rm {} \;
    find /mnt/disk640/ -type f -name "*\$DATA" -exec rm {} \;
    #-exec 要執行的指令
    #{} 每個使用find找到的檔案
    #\; 表 -exec 指令結束,但分號 ; 為特殊字元,故前置倒斜線
  • 停止自動掛載
