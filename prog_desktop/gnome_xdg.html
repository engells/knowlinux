Thems

  • 一般性作法
    1. 備份原來的佈景檔案 /usr/share/theme 文件
    2. 下載佈景套件，可以到 http://www.linuxidc.com/theme/index.htm 尋找
    3. 解壓縮佈景套件，然後把文件拷貝到 /usr/share/theme 目錄下
    4. 在GNOME tweak tool 下選擇該佈景。
  • 安裝 elementary 佈景主題
    1. sudo tar -xzvf Mac-Lion-Cursors.tar.gz -C /usr/share/icons
    2. tar -xzvf Mac-Lion-Theme.tar.gz -C ~/.themes
    3. tar -xzvf Mac-Lion-Icons.tar.gz -C ~/.icons
  • 更換開機動畫， 20120524 實測無效
    1. unzip -d /path/extracted/to filename
    2. sudo update-alternatives --install /lib/plymouth/themes/default.plymouth default.plymouth /lib/plymouth/themes/Paw-OSX/paw-osx.plymouth 100
    3. sudo update-alternatives --config default.plymouth, 選擇 paw-osx.plymouth manual mode
    4. sudo update-initramfs -u

XDG(X DESKTOP GROUP)

RHEL 6 和 Ubuntu 採用了 FreeDesktop.org 的規格，其官方網站 http://standards.freedesktop.org/basedir-spec/basedir-spec-latest.html。該 規格定義了 Linux 下基本的 X Window System (X11) 及其他 Unix-like 作業系統的桌面環境，主要目的是為了增加 free software desktop environments 的互通性，這環境簡稱為 XDG(X Desktop Group)。
  • 修改帳戶目錄
    • /home/engells/.config/user-dirs.dirs
  • 在 x-window 開啟後自動執行命令搞(scripts)
    • ~/.xinitrc, ~/.xsession
      .xinitrc/.xsession 使用上比較麻煩。 不論是誰 (GDM/KDM/startx) 把它叫起來的，控制權就不會再回去。所以 .xinitrc 或 .xsession 裡面的最後一句通常是 exec gnome-session 或 exec startkde 或 exec enlightenment 或 exec icewm 之類。總之，就是自己要變身成某個視窗管理員程式 (或內含視窗管理員的某個桌面環境)。
      有網路文章建議，優先使用 ~/.xsession，然後將 ~/.xinitrc 連結至 ~/.xsession。(ln -s ~/.xsession ~/.xinitrc)
    • ~/.xprofile
      由 GDM/KDM 呼叫起來, 比較單純, 裡面就只放要加的命令就好。 .xprofile 執行結束之後，控制權會回到 GDM / KDM，此外 .xprofile 也負責啟動視窗管理員 (window manager)。
  • 在 x-window 開啟後自動啟動應用程式
    在指定目錄修改或是新增 .desktop 檔案：
    • System-wide autostart directories:
      • /etc/xdg/autostart → existed in Ubuntu，有作用
      • /usr/share/autostart → Ubuntu無此目錄
    • User specific autostart directories:
      • ~/.config/autostart => existed in Ubuntu，有作用
      • ~/.kde/share/autostart  => KDE specific
      • ~/.kde/Autostart => KDE specific
    另外 Ubuntu 下可執行 System > Preferences > Startup Applications 執行 GUI 程式進行設定。
  • 登入帳戶設定
    1. 執行 gdmsetup
    2. 修改 /etc/gdm/gdm.conf
      [daemon]
      TimedLoginEnable=true
      AutomaticLoginEnable=true
      TimedLogin=root
      AutomaticLogin=root  =>設定自動登入帳號
      TimedLoginDelay=30
      DefaultSession=gnome
  • 修改 Gnome 右鍵選單
    Ubuntu 下執行 System > Preference > Main Menu，或在終端機執行 alacarte，可以增、改、隱藏、顯示選單，但即使 root 權限也無法刪除選單。
    修改檔案結構依照 Freedesktop 規則，選單是由指定目錄下的 .desktop 檔案及 icon 圖示動態產生：
    • /usr/share/applications/*.desktop 及 ~/.local/share/applications/*.desktop
      .desktop檔案會在Application、System等主選單下出現成為選單項目。/usr下的是系統層級選項，~/.local下的是用戶用 選項。
    • /usr/share/icons 或者 /usr/share/pixmaps
      icons放置目錄
    • /etc/gnome/defaults.list 及 ~/.local/share/applications/default.list
      default.list 檔案保存的是檔案的預設打開程式，比如 txt 檔用什麼程式打開。/etc 下的適用於全系統，.local 下的適用於帳戶個人。
      Gnome 環境下修改檔案的打開方式，選擇右鍵 > 屬性 > 打開方式 > 選擇某一程式，之後在 Nautilus 雙擊檔案時，系統會啟用相應的程式開檔。此法不能刪除或增加選單中的開檔程式。
    • /usr/share/applications/mimeinfo.cache 及 ~/.local/share/applications/mimeinfo.cache
      mimeinfo.cache 檔定義各種檔案類型的打開方式，包含預設程式在內。預設程式出現在滑鼠右鍵選單頂部，其他程式出現在 "open with" 子選單內。
    • /etc/gnome/defaults.list的結構
      [DefaultApplications]
      application/csv=gnumeric.desktop
      application/excel=ooo-calc.desktop
      application/msexcel=ooo-calc.desktop
      application/msword=ooo-writer.desktop
      application/ogg=mplayer.desktop
      application/pdf=evince.desktop
      application/postscript=evince.desktop
      ...
      項目格式為：檔案副檔名/檔案類型=打開這個檔案的項
    • /usr/share/applications/mimeinfo.cache的結構
      [MIME Cache]
      text/plain=vim.desktop;gvim.desktop;madedit.desktop;openoffice.org-writer.desktop;
      ...
      項目格式為：檔案副檔名/檔案類型=項目1;項目2;…項目n;
      各項目中間用 ";" 隔開，不留其餘字元。要修改某類檔案的開檔程式，以 txt檔為例，text/plain預設開檔程式是 vim，其次是 firefox、gvim等
    • /usr/share/applications/*.desktop檔案的結構
      [DesktopEntry]
      Encoding=UTF-8        #字符編碼
      Name=vim        #程式名稱
      MimeType=text/plain    #檔案類型
      Exec=vim %f        #執行的程式，%f表示執行參數
      Type=Application    #類型
      Terminal=true        #是否使用終端
      NoDisplay=true        #是否顯示在gnome選單裏
