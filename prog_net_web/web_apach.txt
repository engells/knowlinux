Apache 虛擬主機
====================
編輯 /etc/apach2/sites-available/www.abc.com.conf
  <VirtualHost *:80>
    ServerAdmin admit@abc.com
    ServerName abc.com
    ServerAlias www.abc.com
    DocumentRoot /webs/abc
    ErrorLog /webs/logs/engells/www.error.log
    CustomLog /webs/logs/engells/www.access.log  combined
    <Directory "/webs/abc">
      AllowOverride All
      Order Allow,Deny
      Allow from All
    </Directory>
  </VirtualHost> apache 2.2 版以下語法

  <VirtualHost *:80>
    ServerAdmin admit@abc.com
    ServerName abc.com
    ServerAlias acct.abc.com
    DocumentRoot /webs/abc
    ErrorLog off
    CustomLog /dev/null combined
    <Directory "/webs/abc">
      AllowOverride All
      Require all granted
    </Directory>
  </VirtualHost> apache 2.4 版以上語法

建立軟連結並重啟 Apache Server
  編輯 /etc/apache2/apache2.conf，加入 /webs 目錄
  cd /etc/apache2/sites-enabled
  sudo ln -s ../sites-available/www.abc.com.conf ./www.abc.com.conf
  sudo a2ensite www.abc.com
  sudo systemctl reload apache2




Apache 目錄權限
====================
http://xxx.xxx.xxx.xxx/test
編輯 /etc/apach2/apach2.conf
<!-- 設定虛擬目錄為 localhost/test/，實際檔案位置在 /webs/test/ -->
Alias /test/ "/webs/test/"
<Directory "/webs/test/">
  Options FollowSymLinks
  AllowOverride None
  Require all granted
</Directory>




Apach2 SSL Settings
====================
vi /etc/apache2/sites-available/default-ssl.conf 
  line 3: ServerAdmin webmaster@srv.world
  line 32: SSLCertificateFile /etc/ssl/private/server.crt
  line 33: SSLCertificateKeyFile /etc/ssl/private/server.key
a2ensite default-ssl
a2enmod ssl
systemctl restart apache2 




Apach2 Auth+LDAP
====================
a2enmod ldap authnz_ldap
vi /etc/apache2/sites-available/auth-ldap.conf 
  https://www.server-world.info/en/note?os=Debian_8&p=httpd&f=10
  https://sysadmin.compxtreme.ro/enable-ldap-authentication-in-apache/
  <Directory /var/www/html/auth-ldap>
    SSLRequireSSL
    AuthName "LDAP Authentication"
    AuthType Basic
    AuthBasicProvider ldap
    AuthLDAPURL ldap://dlp.srv.world/dc=srv,dc=world?uid?sub?(objectClass=*)
    Require ldap-filter objectClass=posixAccount
  </Directory>
mkdir /var/www/html/auth-ldap 
a2ensite auth-ldap
systemctl restart apache2 
vi /var/www/html/auth-ldap/index.html # create a test html page
