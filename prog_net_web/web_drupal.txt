Drupal 8 MultiSite 多網站安裝
====================
MultiSite 的好處是模組共用，更新任一網站，其他次網站也都一併更新。若資料庫採獨立建置，就可實現多網站+共用網頁檔案+獨立資料庫的架構，方便更新與維護。

啟用多虛擬主機

新增次網站的根目錄
	mkdir -p /path/to/drupal/sites/www1.abc.com
	mkdir -p /path/to/drupal/sites/www2.abc.com
	mkdir -p /path/to/drupal/sites/www1.abc.com/files && chmod 775 /path/to/drupal/sites/www1.abc.com/files
	mkdir -p /path/to/drupal/sites/www2.abc.com/files && chmod 775 /path/to/drupal/sites/www2.abc.com/files
	mkdir -p /path/to/drupal/sites/www1.abc.com/files/translations && chmod 775 /path/to/drupal/sites/www1.abc.com/files/translations <= 語言包使用
	mkdir -p /path/to/drupal/sites/www2.abc.com/files/translations && chmod 775 /path/to/drupal/sites/www2.abc.com/files/translations <= 語言包使用

設定 /path/to/drupal/sites/sites.php
	mv /path/to/drupal/sites/example.sites.php /path/to/drupal/sites/sites.php
	chmod 664 /path/to/drupal/sites/sites.php
	vim /path/to/drupal/sites/sites.php
		$sites = array(
			‘www1.abc.com’ => ‘www1.abc.com’,
			‘www1.abc.com’ => ‘www1.abc.com’,
		);
		格式為：‘<port>.<domain>.<path>’ => ‘directory’，詳細設定參考 example.sites.php

複製 settings.php 和 services.yml 至次網站目錄
	cp /path/to/drupal/sites/default/default.settings.php /path/to/drupal/sites/www1.abc.com/settings.php
	cp /path/to/drupal/sites/default/default.services.yml /path/to/drupal/sites/www1.abc.com/services.yml
	chmod 666 /path/to/drupal/sites/www1.abc.com/settings.php
	chmod 666 /path/to/drupal/sites/www1.abc.com/services.yml
	cp /path/to/drupal/sites/default/default.settings.php /path/to/drupal/sites/www2.abc.com/settings.php
	cp /path/to/drupal/sites/default/default.services.yml /path/to/drupal/sites/www2.abc.com/services.yml
	chmod 666 /path/to/drupal/sites/www2.abc.com/settings.php
	chmod 666 /path/to/drupal/sites/www2.abc.com/services.yml

開啟瀏覽器，輸入次網站的網址，例如：www1.abc.com，會出現 Drupal 安裝畫面

修改檔案權限
	chmod 444 /path/to/drupal/sites/www1.abc.com/settings.php
	chmod 444 /path/to/drupal/sites/www2.abc.com/settings.php

以上為「子網域」模式，若採用「子目錄」模式如：http://abc.com/sub1，需將分支網站所在目錄連結至 Drupal document root 目錄。對照 http://xxx.xxx.xxx.xxx/phpmyadmin 的使用方式。
	ln -s /path/to/drupal/ /path/to/www/sub1 <= sub1 為分支網站名稱

URL: http://blog.fhps.tp.edu.tw/fhpsmis/?p=1273




Drupal 7 Multisite
====================
Apache啟用虛擬多網站
	需將網站位置設為 drupal根目錄

新增次網站的根目錄，必須用全網域名稱
	mkdir -p /path/to/drupal/sites/www1.abc.com
	mkdir -p /path/to/drupal/sites/www2.abc.com

次網站根目錄下產生 settings.php，權限設為 666

次網站根目錄下再分別新增資料夾「files」，權限設定為 777 或 775

開啟瀏覽器，輸入次網站的網址，例如：www1.abc.com，會出現 Drupal 安裝畫面

次網站 settings.php，權限設為 444

LEMP 模式架設 Drupal
====================
apt install -y nginx mariadb-server php7.0-fpm php7.0-mysql php7.0-cli php7.0-gd php7.0-pdo php7.0-xml php7.0-mbstring

mysql_secure_installation or sudo mysql -u root -p

/etc/php/7.0/fpm/php.ini => cgi.fix_pathinfo=0
systemctl restart php7.0-fpm

/etc/nginx/sites-available/
nginx -t
systemctl reload nginx

sudo mysql -u root -p
create database website_main default charset utf8 collate utf8_general_ci;
grant usage on *.* to admit@localhost identified by 'admit@drupal~2015';
grant all privileges on website_main.* to admit@localhost;
exit

use rrxl
source /path/to/xxx.sql




	
LEPP 模式架設 Drupal
====================
apt install -y nginx postgresql php7.0-fpm php7.0-pgsql php7.0-gd php7.0-pdo php7.0-xml php7.0-mbstring

在 PostgreSQL 建立使用者及資料庫
---------
sudo su postgres
createuser user_name --pwprompt --encrypted
createdb db_name --owner=user_name
exit

解壓縮 Drupal 安裝檔後，需將 Owner 變更更為 www-data
