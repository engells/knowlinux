Virtual Host
====================

建立虛擬主機使用的目錄
----------
mkdir /path/to/xyz.aabbcc.com
mkdir /path/to/xyz.aabbcc.com/logs
chmod -R 755 /var/www/xyz.aabbcc.com
chown -R www-data:www-data /var/www/xyz.aabbcc.com <= www-data 為 nginx 預設使用者，詳 /etc/nginx/nginx.conf


建立虛擬主機設定檔
----------
編輯 /etc/nginx/sites-available/xyz.aabbcc.com
	server {
		listen 80;
		listen [::]:80;

		root /path/to/xyz.aabbcc.com;
		index index.php index.html index.htm;
		server_name xyz.aabbcc.com www.xyz.aabbcc.com;

		location / {
            try_files $uri $uri/ =404;
			error_page 404 = @drupal;  <= Clean URLs 需要此句
		}

		location @drupal { <= Clean URLs 需要此段
			rewrite ^(.*)$ /index.php?q=$1 last;
		}

        location ~ \.php$ {
                include snippets/fastcgi-php.conf;
                fastcgi_pass unix:/run/php/php7.0-fpm.sock;
        }

		access_log /var/www/xyz.aabbcc.com/logs/access.log;
		error_log /var/www/xyz.aabbcc.com/logs/error.log;

	}


連結設定檔至 /etc/nginx/sites-enabled/
	ln -s /etc/nginx/sites-available/xyz.aabbcc.com /etc/nginx/sites-enabled/


另參閱：https://www.nginx.com/resources/wiki/start/topics/recipes/drupal/




修改及重啟 nginx
----------
編輯 /etc/nginx/nginx.conf
	啟用 server_names_hash_bucket_size 64;

重啟 nginx
	sudo nginx -t
	sudo systemctl restart nginx

編輯客戶端 /etc/hosts 增加虛擬主機網址
	xxx.xxx.xxx.xxx	xyz.aabbcc.com	www.xyz.aabbcc.com

*****




啟用 PHP 支援
====================
編輯 /etc/nginx/sites-available/xxxx.conf
	fastcgi_pass unix:/run/php/php7.0-fpm.sock;

編輯 /etc/php/7.0/fpm/php.ini
	cgi.fix_pathinfo=0

編輯 /etc/php/7.0/fpm/pool.d/www.conf
	listen = /run/php/php7.0-fpm.sock <= 與 /etc/nginx/sites-available/xxxx.conf 之 fastcgi_pass 之設定一致

重啟 php7.0-fpm 及 nginx
	sudo nginx -t
	sudo systemctl restart php7.0-fpm
	sudo systemctl reload nginx
