語系設定
====================
加入新語系摘要步驟
----------
在 Ubuntu 安裝新的語系
  加入語系來源
    touch /var/lib/locales/supported.d/locale
    echo 'zh_TW.UTF-8 UTF-8' >> /var/lib/locales/supported.d/locale
  安裝新的語系
    locale-gen zh_TW.UTF-8

在 Debian 安裝新的語系
  安裝套件支援語系的設定
    apt-get install locales
  使用圖形化工具安裝新的語系
    dpkg-reconfigure locales
      # 此工具對應至 /etc/locale.gen(語言選擇列表步驟)，/etc/default/locale(設定預設語言步驟)
  手動安裝新的語系
    編輯 /etc/locale.gen 與 /etc/default/locale
      把要產生的 locale 前面的 # 去掉
    安裝新的語系
      locale-gen
    更新預設語系
      update-locale
    設定環境變數
      export LANG=en_US.UTF-8
      export LANGUAGE=
      export LC_ALL=
    重開機或重登入 X 環境


語系相關設定檔
----------
Ubuntu 
  /etc/default/locale => 系統語系設定，Ubuntu 的內容為 LANG="en_US.UTF-8"
  /var/lib/locales/supported.d/* => locale-gen設定檔
  /usr/lib/locale/<locale-name>/ => 已安裝之語系
  /usr/share/i18n/SUPPORTED => 所有支援之語系

其他 Distribution 使用檔案
  /etc/local.gen → Debian，locale-gen設定檔
  /etc/environment → Ubuntu中只設定 $PATH
  /etc/sysconfig/i18n → Red Hat


概說
----------
Locale 是軟體執行時的語言環境，其代碼包括語言(Language)、地域(Territory)和字元集(Codeset)。組合格式爲：語言_地域.字元集。
繁體中文語系代碼為：zh_TW.UTF-8。系統預設的語系為 C 或是 POSIX。
Locale 大多用在libc-based的應用程序上。
Debian 系列 Linux，編譯安裝好的語系位於 /usr/lib/locale/
語系相關變數值，包括：
  LANG：預設語系，效力最低，如果有設其他變數，則其他變數優先使用。如果沒有安裝 locales，預設的 LANG 通常稱為 C 或 POSIX
  LANGUAGE：只有少數程式會用到
  LC_CTYPE：字元分類和處理，一般的輸入法只要設定這個就能用
  LC_NUMERIC：處理非金錢相關的數字格式
  LC_TIME：處理時間、日期等
  LC_COLLATE：處理字元比較和排序
  LC_MONETARY：處理金錢相關的格式和符號
  LC_MESSAGES：處理顯示的格式，訊息想看中文要設定這個
  LC_PAPER
  LC_NAME
  LC_ADDRESS
  LC_TELEPHONE
  LC_MEASUREMENT
  LC_IDENTIFICATION
  LC_ALL：最強效設定，除 $LANGUAGE 外取代以上所有的語系變數


指令
----------
export
  查詢所有環境參數，包括 $LANG
locale
  查詢 locale 相關變數值
locale -a
  查詢已安裝語系，可對照 /usr/lib/locale/<locale-name>
locale-gen zh_TW.UTF-8
  新增繁體語系
dpkg-reconfigure locales
  設定 $LANG。man page 或其他文字通常是以 en 或 en_US 顯示。


語系變數設定範例
----------
export LC_ALL=zh_TW.UTF-8
export LANG=zh_TW.UTF-8
export LC_CTYPE=zh_TW.UTF-8
  在這裡設定 LANG 和 LC_CTYPE 都是沒意義的，因為LC_ALL 已經設定了

export LANG=zh_TW.BIG5
export LC_ALL=zh_TW.UTF-8
  在這裡 LANG 無效

export LANG=zh_TW.BIG5
export LC_CTYPE=zh_TW.UTF-8
export LC_MESSAGES=zh_TW.UTF-8
  顯示輸入都是用 UTF-8, 其他的東西則是 big5

export LANG=POSIX
export LC_CTYPE=zh_TW
  全部都是美式英文, 但可以輸入繁體中文

export LC_MESSAGES=zh_TW
  通常設定這個之後,軟體的介面和選單就會用中文(如果有支援的話)，例外是 openoffice, 還要設定LANGUAGE，代碼:

export LANGUAGE=zh_TW

設定字體
====================
繁體中文字體設定檔：
  目錄 /etc/fonts/conf.d/ 或 ~/.config/fontconfig/fonts.conf，後者可能需要手動建立
  檔案 69-language-selector-zh-tw.conf 字體有三種：serif、sans-serif 及 monospace。各字體有單獨之區段，各區段所列出之字型，即為引用順序。
  檔案 65-nonlatin.conf 可設定中文以及其他非拉丁字母對應字型，不確定與 69-language-selector-zh-tw.con 的適用順序。 

字型顯示
  建議系統字型使用蘋果黑體、蘋果儷黑體或微軟正黑體。
  除系統字型外，FireFox、Chrome、MadEdit、Gedit等均可個別設定字型

Linux 以 fontconfig 套件來管理跟控制字型，可使用 fontconfig 程式，自行設定螢幕顯示字型繪製的方法，或轉換字體格式為 ttf

X-Window Fonts Aliases
====================
指令
----------
/etc/fonts/conf.avail/30-cjk-aliases.conf
/etc/fonts/conf.d/30-cjk-aliases.conf，由 /etc/fonts/conf.avail/30-cjk-aliases.conf 連結而來

sudo ln -s /etc/fonts/conf.avail/30-cjk-aliases.conf  /etc/fonts/conf.d/30-cjk-aliases.conf
  詳細範例請參閱相關網址，設定字體別名簡單範例如下：
    <!--增加一段 alias，指定文泉驛正黑(WenQuanYi Zen Hei)作為 SimHei 的替代字體-->
    <alias>
      <family>SimHei</family>
        <accept>
          <family>WenQuanYi Zen Hei</family>
        </accept>
    </alias>


相關設定檔
----------
字型說明
  黑體
    可設定文泉驛正黑體作為黑體的替代字型。
    目前 Ubuntu 與大多數的 Linux 桌面環境，皆使用文泉驛正黑體作為預設字體。可以作為微軟 Windows 系統的正黑体(SimHei), 微軟正黑體與微軟雅黑體的替代字體。微軟雅黑體是微軟 Windows Vista 簡體中文版的默認字體。微軟正黑體是微軟 Windows Vista 繁體中文版的預設字體。

  楷書
    可設定全字庫正楷體作為標楷體(DFKai-sb)的替代字型。
    Ubuntu 內建提套 AR PL UKai TW 楷體，是文鼎公司捐贈出來的楷書字體。然而，該字體不符合中華民國教育部頒標準字體，嚴格說來，不適合作為「標楷體」的替代。
    標楷體的英文名稱是 DFKai-sb ，為 Windows XP 繁體中文版的預設字體之一。 只有 Windows XP 繁體中文版認「標楷體」這個名稱，其他語系的 Windows 認的是 DFKai-sb 這個名稱。標楷體是微軟向華康公司購買的楷書字型，支援 Unicode ，同時提供簡、繁兩種字體。此外，其繁體字體符合中華民國教育部頒標準字體，故名為「標楷體」。

  明體與宋體
    可設定全字庫正宋體作為細明體(MingLiu)的替代字型。
    Ubuntu 內建提供一套 AR PL UMing 宋體，是文鼎公司捐贈出來的字體。該字體不符合中華民國教育部頒標準字體。
    細明體的英文名稱是 MingLiu ，為 Windows XP 繁體中文版的預設字體之一。 只有 Windows XP 繁體中文版認「細明體」這個名稱，其他語系的 Windows 認的是 MingLiu 這個名稱。細明體是微軟向華康公司購買的字體，支援 Unicode ，同時提供簡、繁兩種字體。
