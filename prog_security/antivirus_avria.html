Install Antivir on Ubuntu

解壓後，終端中輸入：
CD 檔所在目錄（比如：CD /home/jkl/antivir-workstation-pers-2.1.12-19）
sudo ./install
根據提示完成安裝即可，下遍是大概：

然後，注意哦！默認下，您和 root 都不能使用 AntiVir。須在 系統 → 系統管理 → 使用者及群組 → 群組 >antivir → 屬性 → 群組成員加入 root和 您的用戶名
用法是：antivir [參數] [路徑[\*.副檔名]] [*.副檔名]
參數有：
--help .......... 顯示幫助文本 (縮寫: -h 或 -?)
--scan-mode=<mode> 可供使用的掃描模式有“extlist(副檔名列表)”，“smart(智慧)”，“all(全部副檔名)”：
在extlist模式下，antivir根據檔的副檔名決定是否對其進行掃描，
在smart模式下，antivir根據檔的名字或者內容決定是否對其進行掃描，
在all模式下，antivir掃描所有的檔。
--allfiles ...... 等價於--scan-mode=all
--version ....... 顯示版本資訊
--info .......... 顯示所有可以識別的模式。(譯注：這句話很籠統，觀察了該選項的輸出，發現所謂的可以識別的模式包括支援的壓縮包的格式、檔類型，可以識別的病毒等)
--update ........ update antivir
--update ........ 升級。(譯注：包括病毒庫和程式檔)
--check ......... used with --update to check for updates
--check ......... 和選項--update一同使用，用於檢查有無可用更新
--temp=<dir> .... 設定存放暫存檔案的目錄
--pid-dir=<dir> . 設定存放PID檔的目錄
--home-dir=<dir> 指定可執行檔、病毒庫檔和key檔的存放位置
-C <filename> ... 設定配置檔的名稱
-s .............. 掃描子目錄
--scan-in-archive 掃描壓縮檔
-z .............. 等價於--scan-in-archive (scan in archives, too)
--archive-max-size=N, --archive-max-recursion=N, --archive-max-ratio=N, --archive-max-count=N
設定反拒絕服務的閾值：避免掃描符合以下條件的壓縮包：禁止掃描超過指定大小，或者超過指定遞迴層級，或者超過指定壓縮率，或者壓縮包內含超過指定個數的 檔。前3個選項的N如果取0，則意味著沒有限制。(譯注：病毒可以產生一個超級難解開的壓縮包，然後拋給病毒掃描器，借此消耗系統的記憶體或CPU，從而 導致掃描器進程回應緩慢甚至崩潰，病毒得以脫身。這種超級難解開的壓縮包有一些共同特徵，它可能非常大，或者N層遞迴壓縮，或者壓縮比率非常高，或者壓縮 包的某個壓縮層級裏含有異常多的檔。通過給掃描器設定一定的閾值，可以防止這種攻擊)
--scan-in-mbox .. 掃描郵件目錄(可能很耗時！)
--heur-macro .... 啟動巨集病毒啟發
--heur-nomacro .. 禁用巨集病毒啟發
--heur-level=N .. 設定啟發式級別，0為關閉，1-3=低啟發-高啟發
-nolnk .......... 不追蹤符號鏈結
-onefs .......... 跟蹤鏈結，但不跨越不同的檔系統
-noboot ......... 不檢查引導記錄
-nombr .......... 不檢查主引導記錄
-nobreak ........ 不理會Ctl-C和Ctrl-Break
-nodef ......... 不檢查指定的檔類型
-cf<filename> ... 啟動迴圈冗餘校驗檢查，並指定資料庫名
-cv ............. 計算整個檔長度的迴圈冗餘校驗值(默認只計算頭16k的內容)
-cn ............. 插入新檔案名到資料庫中
-cu ............. 重新計算迴圈冗餘校驗值，並更新資料庫
-v .............. 徹底掃描(較慢，並且可能有誤報)
-nopack ......... 不在檔包內掃描
-e [-del | -ren] 如果能修復，那麼修復感染檔
如果使用了[-del]選項，則無法修復的檔將被刪除
如果使用了[-ren]選項，則無法修復的檔將被重命名
-ren ............ 重命名感染檔
-del ............ 刪除感染檔
--moveto=<dir> .. 隔離感染文件
-dmdel .......... 刪除含有可疑巨集病毒的文檔
-dmdas .......... 如果文檔中有一個宏可疑，那麼刪除該文檔中所有的宏
-dmse ........... 如果發現巨集病毒，則設定退出碼為101
-r1 ............. 僅記錄感染和警報
-r2 ............. 並且記錄所有掃描過的路徑
-r3 ............. 並且記錄所有掃描過的檔
-r4 ............. 選擇詳細記錄模式
-rs ............. 選擇單行資訊模式
-rf<filename> ... 命名日誌檔
%d = 日, %m = 月, %y = 年 (日月年均為2位元數字)
-ra ............. 向已經存在的日誌檔追加新資料
-ro ............. 覆蓋已經存在的日誌檔
-q .............. 靜默模式
-lang[:|=]DE .... 使用德文介面
-lang[:|=]EN .... 使用英文介面
-once ........... 1天內只運行一次
-if<dateiname> .. 使用指定的ini檔
--with-<type> ... 同時檢測其他惡意軟體(不是病毒，但並非用戶想要的)
類型包括"dial(撥號程式)", "joke(玩笑程式)", "game(遊戲)",等
--with-alltypes代表所有惡意軟體類型
--without-<type> 含義同--with-<type>選項，但作用相反----不檢查該類型的惡意軟體
--alltypes ...... 等價於--with-alltypes (此選項已作古)
--alert-urls=<yes|no> 為每個警報列印URL，以便獲得進一步的資訊
--warnings-as-alerts 如果發現一個感染檔，並已經發出警報，則以相應退出碼退出
--exclude=<file> 排除檔或目錄
--log-email=<addr> 同時以email形式發出掃描報告
@<rspfile> ...... 從<rspfile>文件中讀取參數
每個選項各占獨立一行

退出碼列表：
0: 正常返回，無任何發現及錯誤。
1: 發現感染檔或引導記錄
2: 在記憶體中發現警報
3: 發現可疑文件
100: 僅顯示幫助文本(而未進行任何掃描)就退出了。(譯注：見於執行antivir --help後)
101: 在一個文檔檔中發現一個巨集病毒
102: 選項-once已啟動，今天已經運行過antivir了
200: 記憶體不足，程式異常終止
201: 未發現指定的回應文件
202: 在指定的回應檔中發現另一個@<rsp>選項
203: 無效選項
204: 無效目錄
205: 無法創建日誌檔
210: 找不到必須的dll檔
211: 自檢失敗，程式異常終止
212: 無法讀取antivir.vdf文件
213: 初始化時出現錯誤
214: 無法找到key文件
