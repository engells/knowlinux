指令示例：
====================
在虛擬機安裝作業系統
  kvm -enable-kvm -bios /usr/share/ovmf/OVMF.fd vdisk.img -cdrom /path/to/boot-media.iso -smp 2 -boot d -m 512
   -enable-kvm: 啟用 kvm 功能
   -bios: 指定 bios 檔案，本例 OVMF.fd 為 UEFI，virtsh 使用 /usr/share/OVMF/OVMF_CODE.fd 檔案作為 UEFI
   -cdrom: 掛載光碟印象檔
   -smp: CPU 核心數
   -boot: 選擇開機方法，d 是指光碟開機
   -m: 記憶體大小，單位 MB

由硬碟分割啟動虛擬機
  kvm -monitor stdio -m 1024 -vga -kernel vmlinuz -initrd initrd.img -append 'root=/dev/sda' /dev/sdz99

由一整顆硬碟或隨身碟啟動虛擬機
  kvm -monitor stdio -m 1024 -vga std /dev/sdz

由 LVM 的 volume 啟動虛擬機
  kvm -monitor stdio -m 1024 -vga /dev/mapper/abc-xyz

由光碟映像檔啟動虛擬機
  kvm -monitor stdio -m 1024 -vga iso_name.iso

由硬碟映像檔般開機：
  kvm vdisk.img -smp 2 -m 512

引導 Ports：
  kvm vdisk.img -smp 2 -m 512 -redir tcp:實體機器的port::虛擬機器的port
   ex：kvm vdisk.img -smp 2 -m 512 -redir tcp:3389::3389

掛載 USB 裝置：
  -usbdevice 'devicename'
   ex：-usbdevice disk:/dev/sdb
   devicename：
    mouse
    tablet
    disk:[format=format]:file
    host:bus.addr
    host:vendor_id:product_id
    ...更詳細資訊請 man kvm

    PS.可以用 lsusb 看裝置在哪
    PS.掛載時可能會碰到權限問題，可以用sudo或是修改該裝置的權限chmod a+wr

與 Virtual Box 相容性問題
====================
KVM 可在 Virtual Box 載入系統核心時執行，但 Virtual Box 不可在 KVM 載入核心時執行。因此較佳的方式是同一時間只執行其中之一。

執行 KVM 時，先停止 Virtual Box，再啟動 KVM
----------
sudo service vboxdrv stop
lsmod | grep vbox
sudo service kvm start ← 需先確定沒有執行 VirtualBox 


執行 Virtual Box 時，先停止 KVM，再啟動 Virtual Box
----------
sudo service kvm stop
lsmod | grep kvm → 確定已經移除 kvm 模組
sudo service vboxdrv start


開機時不載入 kvm 模組
----------
編輯 /etc/init/kvm.conf
  #start on runlevel [2345]


相關指令
----------
modprobe -clfr module_name
  -c：列出目前系統所有的模組
  -l：列出 /lib/modules/$(uname -r)/kernel 目錄下所有的模組
  -f：強制載入模組
  -r：移除模組

產生虛擬機硬碟
====================

產生硬碟映像檔
----------
kvm-img create -f qcow2 x.img 4G
kvm-img convert -O qcow2 x.img y.img
kvm-img info x.img
dd if=/dev/zero of=~/virtual/vd01.raw bs=64k count=4800


fdisk disk image file
----------
modprobe nbd max_part=8
modinfo nbd
kvm-nbd --connect=/dev/nbd0 /path/to/xxxx.qcwo
    # 替代指令：kvm-nbd -c /dev/nbd0 /path/to/xxxx.qcwo
gparted /dev/nbd0
kvm-nbd --disconnect /dev/nbd0
rmmod nbd

mount virtual disk, should have nbd module active, and note that only partitions could be mounted
----------
mkdir /path/to/folder
mount /dev/nbd0p1 /path/to/folder
    # 替代指令：mount -t exfat -o rw /dev/nbd0p1 /path/to/folder，-t vfat | exfat | ntfs | ext4 ...
umount /path/to/folder


使用實體硬碟
----------
產生 NewStorage.xml:
  <disk type='block' device='disk'>
    <driver name='qemu' type='raw' cache='none'/>
    <source dev='/dev/sr0'/>
    <target dev='vdc' bus='virtio'/>
  </disk>
  # vd[X], the disk is connected to a virtio controller
  # sd[X], the disk is connected to a scsi controller
  # hd[X], the disk is connected to an IDE controller


將硬碟設定檔匯入虛擬機
  virsh attach-disk machine_name /dev/sr0 vdc
    # 單純匯入硬碟
  virsh attach-disk machine_name /dev/sr0 vdc --config --type disk
    # 加入 type 參數
  virsh attach-disk machine_name /dev/sr0 vdc --config --type cdrom --mode readonly
    # 加入唯讀光碟


注意：
  至此在虛擬機產生 /dev/vdc 的裝置，其實體位置對應至 Host 的 /dev/sr0
  將整個硬碟映射至虛擬機，應先將該實體硬碟的 MBR 備份，以預防在虛擬機操作時變更到實體硬碟原本的 boot-loader
    # dd if=/dev/sda of=sda.mbr bs=446 count=1


或是在 /etc/libvirt/qemu/<your-vm>.xml 組態檔的 <device> section 增加：
  <disk type='block' device='disk'>
   <driver name='qemu' type='raw'/>
   <source dev='/dev/md/storage'/>
   <target dev='vdb' bus='virtio'/>
  </disk>
執行指令  virsh define /etc/libvirt/qemu/guest_name.xml，使上述修改生效。實做可行。



利用 kvm 虛擬技術將 Linux 安裝於實體硬碟
====================
備份 MBR (非必要)
  sudo dd if=/dev/sda of=sda.mbr bs=446 count=1


以 kvm 載入 Linux 安裝光碟, 開始進行安裝作業
  sudo kvm -hda /dev/sda -cdrom /path/to/fedora18-live-cd.iso -smp 2 -m 1024 -usb -usbdevice tablet -boot d
    # -hda 指向要安裝的硬碟代號（如: /dev/sda），不要指向分割區（如: /dev/sda8）
    # -boot d 強制以光碟開機，否則會跑硬碟的 Bootloader
    # -usb -usbdevice tablet 讓虛擬機器的滑鼠游標與本機游標同步，若以文字介面進行安裝可免用這項設定
    # 選擇自訂安裝, 將 / 安裝於指定分割區（如: /dev/sda8）
    # 把 Bootloader 安裝於上述分割區


安裝後, 在 GRUB 2 選單增添或修改選項
  vi /etc/grub.d/40_custom
    menuentry 'Ubuntu (sda8)' {
      set root='(hd0,8)'
      chainloader +1
    }
  grub2-mkconfig -o /boot/grub2/grub.cfg


把 Bootloader 另外寫入磁碟分割區 (first sector of boot partition)
  grub2-install --force /dev/sda7
    目的: 若 MBR 的 Bootloader 毀損或改寫，仍可用其他 Linux 系統的 GRUB 以 chainloader +1 方式啟動本分割區的 Linux 系統


若安裝時不小心把 Bootloader 寫入 MBR, 想改回原設定
  方式一：寫回備份內容
    dd if=sda.mbr of=/dev/sda
  方式二：以目前設定重新寫入 MBR
    grub2-install /dev/sda

  http://jamyy.us.to/blog/2013/01/4431.html




建置及調整 KVM 環境
====================
安裝
----------
kvm-ok
apt install kvm kvm-ipxe virt-manager
  kvm：kvm 必要套件，一般會同時安裝 qemu-common 及 bridge-utils 套件
  kvm-ipxe：提供 PXE 開機 ROM，供網路開機使用
  virt-manager：圖形管理界面，本身是 libvirt-bin 套件的強化，故安裝時會一併安裝 libvirt-bin、virtinst 套件


檢查是否安裝成成功
----------
lsmod | grep kvm    # 若未出現回報資料，則 kvm 未安裝或未執行
modprobe -l | grep kvm
ls -la /usr/bin | grep kvm


以登入帳號執行 virt-manager
----------
將登入帳號名稱，加入 kvm 及 libvirtd 這二個群組 libvirt libvirt-qemu
  sudo adduser `id -un` kvm && sudo adduser `id -un` libvirt && sudo adduser `id -un` libvirt-qemu
編輯 /etc/libvirt/libvirtd.conf
  設定：unix_sock_rw_perms = "0777"
視情況重新登錄


關閉 libvirt 產生的 virbr0 網卡
----------
virsh net-destroy default
virsh net-undefine default
service libvirt-bin restart
ifconfig


支援 UEFI Fireware
----------
安裝 UEFI firmware for virtual machines
  apt install ovmf
將虛擬機設為 UEFI 模式
  vim /etc/libvirt/qemu/virtual_machine
  在 <os> 區段加入 <loader readonly='yes' type='pflash'>/usr/share/OVMF/OVMF_CODE.fd</loader>


libvirt 網路機制
----------
預設產生 virbr0 虛擬橋接器，供 NAT 虛擬網路模式使用
虛擬機所用之 NAT 模式網路卡，依序為 vbet0、vbet1﹍
之後建立 virbr[x] 虛擬橋接器，供 isolated 等非 NAT 之虛擬網路模式使用


設定檔
----------
虛擬機資料：/etc/libvirt/qemu/
網路資料：/etc/libvirt/qemu/network




virsh
====================
常用指令
---------
/etc/libvirt/qemu/              # 虛擬機器配置檔案預設目錄
/var/lib/libvirt/images/        # 預設映象檔案位置 

virsh list --all								# 檢視虛擬機器狀態
virsh autostart virtual_machine_name			# 設定物理機開機自動啟動虛擬機器
virsh dumpxml virtual_machine_name > file_name.xml		# 匯出虛擬機器配置檔案
virsh start virtual_machine_name				# 開機
virsh shutdown virtual_machine_name				# 關機（虛擬機器需開啟acpid服務）
virsh destroy virtual_machine_name				# 強制開機（斷電）
virsh suspend virtual_machine_name				# 掛起虛擬機器
virsh resume virtual_machine_name				# 恢復掛起的虛擬機器
virsh undefine virtual_machine_name				# 刪除一個虛擬機器（會一起刪除配置檔案，映象不刪除）

virsh define /etc/libvirt/qemu/virtual_machine_name.xml		# 重新定義一個虛擬機器
virsh edit virtual_machine_name								# 編輯配置檔案

virsh snapshot-list virtual_machine_name					# 檢視虛擬機器快照
virsh snapshot-create virtual_machine_name					# 建虛擬機器快照（建快照映象格式要為qcow2格式）
virsh snapshot-revert virtual_machine_name snapshot_name	# 還原虛擬機器快照
virsh snapshot-delete virtual_machine_name snapshot_name	# 刪除虛擬機器快照

virsh attach-disk virtual_machine_name /path/to/image_file vdb --cache=none --subdriver=qcow2			# 線上新增虛擬機器映象硬碟
virt-clone -o original_virtual_machine_name -n new_virtual_machine_name -f  /path/to/new_image_file     # 克隆虛擬機器

qemu-img create -f qcow2  /path/to/image_file 20G					# 新建虛擬機器映象磁碟
qemu-img info virtual_machine_name									# 檢視虛擬機器映象格式
qemu-img convert -f raw -O qcow2 image_file.img image_file.qcow2	# 轉換虛擬機器映象格式

virsh nodeinfo				# 列出主機(node host)相關資訊
virsh nodedev-list --tree	# 列出主機(node host)所有硬體裝置(device)
virsh net-list --all		# 列出所有虛擬網路(virtual network)
virsh net-start network_name	# 啟用虛擬網路，例如 virsh net-start default
virsh net-destroy network_name	# 停止虛擬網路
virsh net-autostart network_name	# 設定虛擬網路自動啟動
vim network_name.xml && virsh net-define network_name.xml	# 新增不同設定的虛擬網路，xml 檔的格式參考 http://libvirt.org/formatnetwork.html
virsh net-edit network_name			# 輯虛擬網路設定，預設以 vi 開啟設定檔，如要變更編輯器，請變更 $VISUAL 或 $EDITOR 兩個環境變數 
virsh net-undefine network_name		# 刪除虛擬網路，Debian 環境中，虛擬網路的設定檔存在 /etc/libvirt/qemu/networks/ 
virsh net-dumpxml network_name		# 匯出虛擬網路的 XML 內容，例如 virsh net-dumpxml default > my_default_backup.xml






安裝新虛擬機
----------
virt-install \
	--connect=qemu:///system --name cmdtest --ram 2048 --vcpus=1 --os-type=linux --accelerate --hvm \
	--disk path=/home/mine612/qiuyangjun/ubuntu-16.04/cmd/cmdtest.img,size=8,bus=virtio \
	--cdrom /home/mine612/qiuyangjun/ubuntu-16.04/cmd/ubuntu-16.04-desktop-amd64.iso  \
	--network bridge=ovs,model=virtio \
	--graphics vnc

	--name		# 虛擬機名字
	--ram		# 分配給虛擬機器的記憶體，單位 MB
	--vcpus		# 分配給虛擬機器的 cpu 個數
	--os-type=	# 作業系統類型 [ex: linux]
	--os-variant	# 作業系統的版本名稱 [ex: ubuntu jaunty]
	--accelerate	# KVM 加速器
	--hvm		# 全虛擬化，hvm 與 paravirt 擇其一
	--paravirt	# 半虛擬化，hvm 與 paravirt 擇其一
	--disk		# 指定虛擬機器 img 檔案路徑，如果虛擬機器使用 lvm 分割槽，這裡就指向到 lvm 的分割槽就行
	  size		# 虛擬機器檔案大小，單位 GB
	  bus		# 虛擬機器磁碟使用的匯流排型別，為了使虛擬機器達到好的效能，這裡使用virtio
	  cache		# 虛擬機器磁碟的 cache 型別
		--file	# 另一種方式表達虛擬硬碟的路徑
		--file-size	# 另一種方式表達虛擬硬碟的大小，單位 GB
	--cdrom		# 指定安裝檔案的全路徑
	--network
	   bridge	# 指定橋接網絡卡
		--bridge=br0	# 另一種方式表達橋接網絡卡
	   model	# 網絡卡模式，這裡也是使用效能更好的 virtio
	--graphics	# 圖形引數
		--vnc 
	--noautoconsole
	--debug
