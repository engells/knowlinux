Container 指令
====================
查詢 container
	docker ps 或 docker ps -a
	docker inspect -f "{{ .State.Pid }}" container_id

產生並啟動 container
	docker run -it ubuntu:16.04 /bin/bash
		-i：以互動模式啟動 container
		-t：產生虛擬終端機
		ubuntu:16.04：以 repository:tag 格式表示作為基底的 image
		bin/bash：表示在 container 要執行的程式

	docker run -d --name dbmaria -p 80:80 -v /home/db/maria:/var//lib/maria -e MYSQL_ROOT_PASSWORD='123' -e MYSQL_DATABASE='ayu' ubuntu:16.04
		-d：背景執行 container，會傳回 container id
		--name dbmaria：將 container 命名為 dbmaria
		-p 80:8080：將 host 的 80 port 映射為 container 的 8080 port，可使用多個 -p 參數
		-v /home/db/maria:/var//lib/maria：將 host 的 /home/db/maria 掛載為 container 的 /var//lib/maria，可使用多個 -v 參數
		-e MYSQL_ROOT_PASSWORD='123'：設定環境參數，資料庫管理密碼為 123
		-e MYSQL_DATABASE='ayu'：設定環境參數，資料庫為 ayu
		--net=net_name：指定使用 net_name 網路，該網路由 docker network create 指令建立
		--rm
		--restart=xxx

	docker run -d --name dbshare -v /host/folder:/container/folder image_ref
	docker run -d --name db01 --volumes-from dbshare image_ref
	docker run -d --name db02 --volumes-from db01 image_ref

  docker run --privileged image_name
    --privileged：以 privileged 模式產生 container

啟動已存在之 container
	docker start

進入 container
	docker exec -it [container id or name] /bin/bash
	docker attach [container id or name]
	nsenter --target PID --mount --uts --ipc --net --pid

停止 container
	docker stop [container id or name]

導出 container 並回存為 local image
	docker export [container id or name] > file.tar
	cat file.tar | docker import - ubuntu:16.04

刪除 container
	docker rm [container id]
		僅能刪除停止運作之 container，可加入 -f 參數強制刪除運作中的 container

	docker rm -f [container name]

	docker rm $(docker ps -a -q)




Image 指令
====================
查詢 repository images
	docker search ubuntu:16.04

查詢 local images
	docker images

下載 image
	docker pull ubuntu:16.04
	docker pull centos:latest

登錄 Docker Repository
	docker login

刪除 image
	docker rmi ubuntu:16.04
	docker rmi $(docker images -q -f "dangling=true")
		-q：--quiet
		-f：--filter
		"dangling=true"：表示無標籤之 local image

將 image 備份為檔案及回存至 local image pool
	docker save -o xenial.tar ubuntu:16.04
	docker load --input xenial.tar 或 docker load < xenial.tar

修改 image 的 tag
	docker tag




Network bridge
====================
使用 docker 預設橋接器
---------------
建立橋接網絡(bridge)
  docker network create -d bridge bridge_name
	  # -d => --driver

編輯預設橋接器組態
  edit /etc/default/docker
	  DOCKER_OPTS="--dns 8.8.8.8 --dns 8.8.4.4 --bip=192.168.20.0/24 -b=br20"
	  --dns：指 docker0 所提供 DNS 位址
	  --bip：設定的網段
	  -b：指定橋接網路介面


使用自設橋接器
---------------
停止 docker 服務，並刪除 docker 預設的橋接器
  service docker stop or systemctl stop docker
  ip link set dev bridge0 down or ifconfig bridge0 down && ifdown bridge0
  brctl delbr bridge0

建立一個橋接器 dockbr0
  brctl addbr dockbr0
  ip addr add 192.168.5.1/24 dev dockbr0
  ip link set dev dockbr0 up

檢視確認橋接器建立並啟動
  ip addr show dockbr0

設定 Docker 服務， 預設橋接到建立的橋接器上。
  echo 'DOCKER_OPTS="-b=dockbr0"' >> /etc/default/docker
  service docker start or systemctl restart docker

Unpriviledge mode
====================
usermod -aG docker user_name
  # 若未建有 docker 群組，執行指令： groupadd docker
	# 若未建有 user 帳戶，執行指令：：useradd -m -s /bin/bash user_name && passwd user_name
logout and login 或執行指令： newgrp docker 轉換為 docker 群組身份

check whether you are running a container in privileged mode, use the command:
  docker inspect --format='{{.HostConfig.Privileged}}' container_id




Linux 執行 Docker GUI 的中文輸入法啟用方式
====================
略過 Docker 要 run 哪種 GUI ，只處理跑起來後的中文輸入法問題，主要是加上下述 -e 的那幾行：
    docker run --rm \
    -e XMODIFIERS="@im=ibus" \
    -e QT_IM_MODULE="ibus" \
    -e GTK_IM_MODULE="ibus"
若使用其它輸入法的，就把 ibus 修改成自己的輸入法，例如：fcitx、scim、gcin...

資料來源： https://blog.longwin.com.tw/2020/12/linux-docker-gui-chinese-input-enable-2020/




Install Docker
====================
Ubuntu from docker.com
----------
sudo apt update && sudo apt install apt-transport-https ca-certificates	software-properties-common
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
sudo apt-key fingerprint 0EBFCD88
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
sudo apt update
#apt install linux-image-extra-$(uname -r) linux-image-extra-virtual
sudo apt install docker-ce			# apt-cache madison docker-ce 可查詢軟體版本。docker-engine 不需安裝。
sudo systemctl start docker
sudo systemctl enable docker    # 開機時啟動 Docker 服務
sudo docker version
sudo docker info     # 


Ubuntu from apt
----------
apt remove docker docker-engine docker.io
apt install docker.io
systemctl start docker
systemctl enable docker	 # optional
docker --version


Centos
----------
yum install -y docker  
systemctl start docker  


Proxmox VE 下的 LXC Ubuntu container 安裝及設定 docker-ce
----------
apt install gnupg2 curl software-properties-common
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(cat /etc/os-release | grep UBUNTU_CODENAME | cut -d '=' -f 2) stable"
apt update && apt install docker-ce

Minimum image file
----------
FROM golang:alpine AS build-env
WORKDIR /app
ADD . /app
RUN cd /app && go build -o goapp

FROM alpine
RUN apk update && \
   apk add ca-certificates && \
   update-ca-certificates && \
   rm -rf /var/cache/apk/*
WORKDIR /app
COPY --from=build-env /app/goapp /app
EXPOSE 8080
ENTRYPOINT ./goapp

習慣上 Docker 的影像（image）名稱會以這樣的格式命名： username/repository:tag

組態檔案： /etc/default/docker
         /etc/docker/daemon.json => { ... "data-root": "/new-path/docker" ... }
         /lib/systemd/system/docker.service
