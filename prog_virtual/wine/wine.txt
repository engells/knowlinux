中文顯示為方塊的解決
-------------------------
常見的說法是，wine 'C:\windows\regedit' → 打開注冊表 → HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\FontSubstitutes → 刪除所有鍵值 → 其實只要刪除 MS Shell Dlg 和MS Shell Dlg2 即可。主要原因為 tahoma 字體在 Linux 下無法顯示

概說
----------------
有時不希望所有程式安裝於一個.wine中。因為有些程式需要特別的運行環境，或者對.wine的更改比較大（經常需要刪除.wine),或者直接影響其它程式運行。

此時需配合 WINE 的組態配置，WINE 預設將配置文件和安裝的 Windows 程序保存在 ~/.wine。這樣的目錄稱為一個「Wine Prefix」或「Wine Bottle」。下文統稱「系統目錄」。每次運行 Windows 程序，包括內置程序如 winecfg時，系統目錄會自動建立或更新。WINE環境下，傳統的 C 槽為  $WINEPREFIX/drive_c
設置 WINEPREFIX 環境變量，可以更改 WINE 系統目錄的位置。如果要讓不同的 Windows 程序使用不同的系統環境或配置，這一變量非常有用。例如，使用 env WINEPREFIX=~/.win-a aa.exe 參數來運行一個程序。另一個使用 env WINEPREFIX=~/.win-b bb.exe 參數，這兩個程序將使用獨立的 C 槽和注冊表配置。

以下命令會建立一個默認的系統目錄，且不啟動任何Windows程序： env WINEPREFIX=~/.customprefix wineboot -u

Q & A
==============================

Q.wine 可以執行所有 windows 軟體嗎？

A.很不幸的，最原始版的 wine，只能執行非常少數的 windows 軟體，通常是一些簡單的小軟體。太複雜的軟體就執行不了，尤其遊戲軟體不修改根本不可能運行。若有軟體不能執行，通常是缺 dll 檔，或者註冊表的原因，使用者必須自己想辦法解決，網路上多半有資料可查，不過英文居多。

另外也有一些輔助程式，幫使用者自動解決這些疑難雜症
例如：（只列幾個比較熱門的）

playonlinux / a frond-end of wine / http://www.playonlinux.com/en/

winetricks / a shell scripts to download and install software / http://wiki.winehq.org/winetricks

PyWinery  / a graphical, easy and simple wine-prefix manager which allows you to launch apps, explore and manage configuration of separate prefixes, graphically and faster than setting variables on linux shell.

Ubuntu 中國論壇的 wine 版，也有很多參考資料 / http://forum.ubuntu.org.cn/viewforum.php?f=121


Q.wine 的執行速度好嗎？若和「虛擬機器」比較呢？

A.事實上，wine 執行軟體的速度很慢，並非 windows 下原生的速度。用 VirtualBox 或 VMware 之類的虛擬機器，直接跑 windows，速度都比 wine 來的快。但 wine 的好處是：省系統資源，有機會運行 3D 遊戲。

用「虛擬機器」光開一個 windows 就要吃掉 512MB 以上，wine 執行一個 windows 軟體，只消耗那個軟體本身吃掉的記憶體，還有 wine 吃一點點而已。此外雖然目前有些「虛擬機器」支援 DirectX (D3D)，但實際上根本無法運行複雜的遊戲，一開就黑屏當機了。而 wine 下是有機會去玩那些複雜的 windows 遊戲的。


Q.wine 本身有版權爭議嗎？是否算盜版？

A.wine 原始版本（從 Synaptic 下載安裝那個），是徹底自由軟體的，沒有版權爭議，其內帶的 dll 檔，是透過逆向工程破解重新撰寫出來的。

但是原始版本能執行的 windows 軟體非常有限。若要執行更多軟體，使用者可能會替換掉 dll 檔或是從 windows 那邊抓檔案過來增加，此舉則有盜版爭議。


Q.上面的 wine 的設定篇步驟中，一定要用那個 Droid Sans Fallback 自由字型嗎？ 能否用其它字型？

A.可以的，只要是內含 utf8/Big5 編碼的「中文字型」就可以。

例如自由字型還有：文泉譯正黑 / http://wenq.org/

用自由字型是為了避免版權爭議，若不在意這點，或有購買字型，也可以拷貝 windows 下的商業中文字型來用。

若要用其它中文字型，只需在 *.reg 檔中，將字型名稱取代替換掉即可。要注意的一點是：「字型名稱」不等於「字型檔名」，在 Ubuntu 下，點兩下 *.ttf 或 *.ttc 字型檔，就會開啟字型預覽程式，可以看到字型的名稱。


Q.wine 執行 windows 軟體，一定要安裝嗎？一定要放 ~/.wine/drive_c/Program Files 裡頭嗎？ 

A.不用，綠色版或 portable 可攜版也可以直接執行，不用安裝也可以跑。上面演示的 uTorrent 運行圖，就是綠色可攜版的。

windows 軟體放在任何地方都可以執行，甚至放在掛載的 NTFS 磁區的軟體也可以直接執行。不一定要放在 ~/.wine/drive_c/Program Files 裡頭。


Q.如果不小心把 ~/.wine 目錄裡的資料弄亂了，怎麼辦？ 

A.直接砍掉重鍊即可。

先關閉所有 wine 的視窗，刪除 ~/.wine。然後開啟 wine 控制台，開啟的瞬間它就會自動重建 ~/.wine 目錄資料到最原始的狀態。

編譯 wine 作者：rossiliu
url:http://www.ubuntu-tw.org/modules/newbb/viewtopic.php?topic_id=12836&viewmode=flat&order=ASC&type=&mode=0&start=0
==============================

sudo apt-get install apt-build	# 安裝apt-build，安裝好之後會詢問使用那種 CPU，要強化到那種等級。我是選擇最強的強化。

deb-src http://wine.budgetdedicated.com/apt intrepid main	# 加入 wine 的 source

sudo apt-build update	# 接下來更新套件庫資訊

sudo apt-get build-dep wine	# 針對 wine 套件編譯，如果有週邊需要的編譯套件，會預先安裝好，避免發生缺乏套件庫而無法安裝的錯誤

sudo apt-build install wine --reinstall	# 漫長的重新編譯後會重新安裝 wine。令人驚訝的是啟動 wine 變得異常的快速，幾近原生程式

sudo apt-get autoremove	# 把剛才安裝編譯所需套件，如果目前沒有其其他軟體用得到，統統移除節省硬碟空間。

sudo apt-build clean-sources	# 將下載下來的 source 清除

sudo apt-build clean-build	# 將編譯過程當中產生的暫存檔案清除

完成。

注意！sudo 不能亂用的, 無論任何情況，都不要用 sudo 或管理員身份執行 wine 或是其相關設定工具如 winetricks、winecfg 等。
